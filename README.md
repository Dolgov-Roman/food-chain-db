# food-chain-db

Проект базы данных PostgreSQL для сети заведений общественного питания (ресторан / суши-бар / кондитерская): проектирование схемы, разграничение доступа, RLS-политики, автоматизация через функции/триггеры/процедуры, индексация для производительности и дополнительные функции (альтернативы бронирования + отчётность по сотрудникам).

## Возможности

### Модель данных
- Города и заведения сети
- Сотрудники, должности и привязка сотрудников к заведениям
- Меню: категории, блюда, состав блюд (ингредиенты)
- Поставщики и складские остатки по заведениям
- Заявки на закупку ингредиентов
- Заказы гостей: позиции заказа, статусы, итоговая сумма
- Отзывы клиентов (оценка + комментарий)

### Безопасность и доступ
- Сняты дефолтные права роли `PUBLIC`
- Групповые роли: `app_admins`, `app_analysts`, `app_managers`, `app_cooks`, `app_waiters`
- Включён Row Level Security (RLS) для таблиц:
  - `customer_order`
  - `order_item`
  - `ingredient_stock`
  - `ingredient_request`
- Операционные роли видят только данные “своих” заведений (через `employee_restaurant` + `get_current_employee_id()`)

### Автоматизация
- Автопересчёт суммы заказа (`customer_order.total_amount`) при изменении `order_item`
- Автоматическое списание ингредиентов со склада при добавлении позиции заказа (с проверкой достаточности остатков)
- Закрытие заказа одним вызовом с записью отзыва: `close_order_with_feedback(order_id, rating, comment)`

### Производительность
- Индексы под частые фильтры и JOIN (заведения, сотрудники, склад, заявки, заказы, позиции, отзывы)

### Дополнительные задания
- `suggest_booking_alternatives(...)` — подбор альтернативного времени или другого заведения в том же городе при перегрузке
- Представление `order_performance` — метрики заказа + вычисляемая длительность выполнения (мин) по времени отзыва
- Функция `employee_performance_report(p_from, p_to)` — отчёт по производительности сотрудников за период

## Установка и запуск

### Требования
- PostgreSQL 13+ (рекомендуется 14/15)
- `psql` (или любой клиент, например DBeaver)

### Установка (через psql)
```bash
createdb food_chain
psql -d food_chain -f sql/00_schema.sql
psql -d food_chain -f sql/01_seed.sql
psql -d food_chain -f sql/02_roles.sql
psql -d food_chain -f sql/03_rls.sql
psql -d food_chain -f sql/04_logic.sql
psql -d food_chain -f sql/05_indexes.sql
psql -d food_chain -f sql/06_extra.sql

### Запуск и проверка
После выполнения скриптов можно проверить работу ключевой логики следующими запросами.

**1) Подбор альтернатив бронирования**
```sql
SELECT *
FROM food_chain.suggest_booking_alternatives(
  p_restaurant_id => 1,
  p_desired_time  => now(),
  p_guests        => 4,
  p_max_orders    => 5
);
